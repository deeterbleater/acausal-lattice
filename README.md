# Acausal Lattice

**Acausal Lattice** is a high-performance Rust implementation of the "Orthogonal Architecture" and "Acausal Ontological Framework" papers. It replaces traditional linear execution loops in AI systems with **State Precipitation** across an **Orthomodular Event Lattice**.

In this system, causality is not a background metric but a partial order derived from structural dependencies. Events (Achronons) precipitate from potentiality into reality only when their prerequisites are met, allowing for massive parallel execution of "spacelike separated" (orthogonal) actions.

## 🌌 Core Concepts

- **The Achronon:** The atomic unit of reality, defined as a triple: Antecedents (dependencies), Orthogonals (independent events), and Transformation (latent state operator).
- **State Precipitation:** The process where Achronons collapse from the "Aion" (the web of potentiality) into the "Registry" (precipitated reality).
- **Tensor Fusion:** Orthogonal events commute. Their transformation matrices are fused into a single tensor contraction ($T_{batch}$), maximizing GPU throughput.
- **Acausal Invariant:** The mathematical guarantee that no two events in a batch have a causal dependency or contend for the same state subspace.

## 🏗️ Architecture

The system is composed of four specialized engines:

1.  **Lattice Topology Engine (LTE):** Manages the Aion using **Roaring Bitmaps** for SIMD-accelerated bitwise eligibility checks.
2.  **Tensor Transformation Engine (TTE):** A GPU-ready engine built on **Hugging Face's `candle`**. It warps an $N$-dimensional latent space using subspace projections.
3.  **Cognitive Context Engine (CCE):** Flattens the lattice state into high-density text prompts for LLMs, prioritizing structural relations over chronological time.
4.  **LLM Integration:** Powered by **Claude 4.6 Sonnet**, providing a dynamic feedback loop where autonomous agents propose new potential realities.

## 🎭 Multi-Agent Simulation

The prototype demonstrates a structural battle between two competing agents:
- **The Architect:** An agent of order, building complex and stable structural expansions.
- **The Disruptor:** An agent of entropy, introducing anomalies and radical causal bifurcations.

## 🚀 Getting Started

### Prerequisites
- **Rust:** (Edition 2021)
- **Anthropic API Key:** Required for dynamic autonomous evolution.

### Installation
1. Clone the repository.
2. Create a `.env` file in the root and add your key:
   ```env
   ANTHROPIC_API_KEY=your_sk_key_here
   ```

### Running the Simulation
```bash
cargo run -p lattice-cli
```
Once the simulation starts, open your browser to **[http://localhost:3000](http://localhost:3000)** to view the live lattice.

## 🛠️ Project Structure
- `crates/lattice-core`: Foundational types and LTE logic.
- `crates/lattice-tensor`: TTE implementation with `candle-core`.
- `crates/lattice-cce`: Logic for state flattening and prompt engineering.
- `crates/lattice-llm`: Anthropic API client for autonomous agents.
- `crates/lattice-daemon`: Axum server and SSE streaming for visualization.
- `crates/lattice-cli`: The main simulation entry point and multi-agent loop.
